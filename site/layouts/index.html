AlphaZero used 5,000 TPUs for self-play data generation and 64 TPU-v2s to train. We use the same network (a 20-block × 256-channel ResNet, ~24M parameters) on one GPU. The number above is the strongest checkpoint we have so far — supervised distillation from Stockfish on 30M positions, evaluated against a calibrated 1,350-Elo opponent. Routine 800-sim eval, no inference-search tricks.

Against a tighter UCI=1,800 opponent (calibrated near our model strength), the same checkpoint scores 2,004 Elo — read that as the most honest current number. Total training cost: about 16 GPU-hours on a single L40S. Roughly four orders of magnitude less compute than the AlphaZero paper.

what we found

Three single-variable ablations on top of the distilled baseline. Each tests whether a different knob controls the Elo ceiling.

network capacity

doubling network depth didn't help

We doubled the ResNet from 20 blocks to 40 blocks (~48M params), kept everything else identical, and trained for the same 20 epochs on the same 5M positions. The 40-block net landed at the same ~1,810 Elo as the 20-block baseline.

Interpretation: with 5M positions of teacher signal, 24M parameters is already in the over-parameterized regime. More capacity doesn't find more structure that isn't there. The bottleneck isn't the network.

eval-side search

+277 elo from search alone, no retraining

The routine evaluation uses 800 MCTS sims/move (AlphaZero's training-time value). We re-ran one checkpoint at 4,000 sims and the same weights scored 2,084 Elo instead of 1,807. 101 wins / 3 draws / 0 losses against the calibrated 1,350-Elo opponent.

{{ partial "search-curve-svg.html" . }}

Interpretation: distilled priors transfer well to deeper inference-time search — exactly Lc0's empirical finding. The "1,800 ceiling" is partly an eval artifact, not a model limit.

data scale

+199 elo from 6× more positions — the winning lever

The baseline used 5M positions subsampled from a 30M-position dataset. We trained on the full 30M (same network, same hyperparameters, even a weaker teacher: Stockfish depth 10 vs depth 15) and the checkpoint at epoch 9 scores 2,009 Elo at UCI=1,800 — +199 over the d15 baseline. Epoch 4 hits 2,084 Elo at UCI=1,350 (101 wins / 3 draws / 0 losses).

The bottleneck on the original baseline was data, not teacher quality or network capacity. 16 h on a g6e.8xlarge, ~$50 of compute. The most cost-effective single change in the project.

how the network plays

At inference time, the trained network is used as a prior over moves and a value estimator for leaf nodes. The actual decision comes from Monte Carlo Tree Search, which expands the most promising lines and reports the visit-count distribution.

{{ partial "mcts-svg.html" . }}

A handful of root-level candidate moves; each sampled deeper based on the network's prior weighted against exploration. After N rollouts the chosen move is whichever was visited most. → how the method works in detail

what's next

The largest remaining gap is that AlphaZero has no teacher — its training targets come from its own MCTS visit counts, not from Stockfish. The Lc0 recipe — distill first, RL second — runs an AlphaZero-style self-play loop on top of the distilled prior. That's running on an EKS pod right now.
